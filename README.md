## 🔧 기술 스택

- **Front-end**
  - **React**: 인터랙티브하고 효율적인 웹 애플리케이션을 만드는 데 최적화된 라이브러리.
  - **TanStack Query**: 서버 상태 관리를 쉽게 도와주는 라이브러리.
  - **React Router**: 페이지 간 네비게이션을 관리하는 라이브러리.
- **Language**
  - **JavaScript**: 프론트엔드 애플리케이션의 주 언어.
- **Deployment**
  - **Vercel**: 웹사이트 호스팅 서비스.
- **Version Control**: Git, GitHub

## Commit 컨벤션

**Commit 태그 종류**

```
docs : README나 WIKI 등의 문서 수정
feat : 새로운 기능 추가
fix : 버그, 오류 수정
modify : 기능 변경, 오타 등으로 인한 코드 수정
refactor : 기능 변경이 일어나지 않는 유지보수 작업 (코드 구조 변경, 불필요한 리소스 삭제, 이름 변경 등)
style : UI/UX, 스타일링 등 디자인 관련 작업
test : 테스트 추가 또는 이전 테스트 수정
chore : 빌드, 패키지 관리, 주석 추가 및 수정 등 기타 작업
```

<br />

**Commit 메시지 형식**

```js
커밋 태그: #이슈 번호 커밋 내용

// 예시
feat: #1 회원가입 기능 완료
```

<br />

## Issue 컨벤션

**Issue 태그 종류**

```
feat: 기능 추가
fix : 에러 수정, 버그 수정
docs : README, 문서
refactor : 코드 리펙토링 (기능 변경 없이 코드만 수정할 때)
modify : 코드 수정 (기능의 변화가 있을 때)
chore : 그 외 작업 내용
```

<br />

**Issue 제목**

```js
커밋 태그: 커밋 내용

// 예시
feat: 회원가입 구현
```

<br />

**브랜치 이름**

```
feat/#이슈 번호-기능 이름

// 예시
feat/#1-login
```
